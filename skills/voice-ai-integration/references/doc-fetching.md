# Shengwang Doc Fetching Guide

## Overview

Use this guide only when the current product module still needs external documentation lookup after the local module docs have been read.

Shengwang docs are fetched via HTTP. The committed indexes under `references/doc-index/` help narrow the target before any fetch.

## Step 1: Use the routed product module first

Read the current product module first. Do not jump here before the module itself signals that local references are insufficient.

## Step 2: Use the bundled static indexes

You already know the product (from the routing step). Use this decision path:

1. Read `references/doc-index/docs.index.md` — find your product section, grab the URI if listed
2. If the URI is not in the MD → load `references/doc-index/shards/{product}.json` and search by slug/kind
3. For SDK class/interface docs → load `references/doc-index/shards/api-ref/{product}-{platform}.json`

These files are generated by the maintainer and shipped with the skill. Do not load the full `docs.index.json` (2MB+) unless the above steps fail.

Use these files to narrow by:
- product
- kind (quickstart, operation, webhook, api-ref, guide, etc.)
- platform (for SDK products: web=javascript, 鸿蒙=harmonyos)

If the bundled indexes do not surface the target, fall back to targeted remote doc fetch or the official doc-site URL pattern.

## Step 3: Extract the document URI

If you need the exact URI, use `references/doc-index/docs.index.json` first. Each record contains:
- `uri`
- `product`
- `platform`
- `section_group`
- `kind`
- `tags`

When the bundled indexes do not expose the exact URI, derive it from the product/platform/path rules below or use the official doc-site URL pattern as a fallback.

## Step 4: Fetch the document

Use the fetch script to get Markdown content:

```bash
bash skills/voice-ai-integration/scripts/fetch-doc-content.sh "docs://default/convoai/restful/get-started/quick-start"
bash skills/voice-ai-integration/scripts/fetch-doc-content.sh "docs://default/rtc/javascript/get-started/quick-start"
```

For ConvoAI, do not start here by default. Inspect the matching sample repo, `agent-server-sdk` on the server side, and `agora-agent-client-toolkit` on the client side when possible first, then fetch REST docs only for missing schemas, vendor parameters, or unsupported operations.

The script handles the query parameter internally via curl, which works in environments
where web fetch tools cannot pass `?uri=...` query strings.

## URI Pattern

```
docs://default/{product}/{platform}/{path}
```

| Product | URI prefix |
|---------|-----------|
| ConvoAI | `docs://default/convoai/restful/...` |
| RTC | `docs://default/rtc/{platform}/...` |
| RTM | `docs://default/rtm2/{platform}/...` |
| Cloud Recording | `docs://default/cloud-recording/restful/...` |
| Token Auth | `docs://default/rtc/{platform}/basic-features/token-authentication` |

## Maintainer Refresh

The bundled indexes are refreshed by maintainers, not by runtime flows.

Maintainers can refresh and rebuild them with:
```bash
bash scripts/fetch-docs.sh
```

Agents should not perform ad hoc mass-search over a raw sitemap file during normal skill execution.

## When to fetch

Fetch for:
- API field details, request/response schemas
- Vendor configurations (TTS, ASR)
- Error codes and meanings
- Any content that may change with doc updates
- ConvoAI REST references only after sample/SDK inspection leaves a gap

Do NOT fetch for:
- Generation rules (field types, naming conventions) — stable, in skill files
- Auth patterns — stable, in [general/credentials-and-auth.md](general/credentials-and-auth.md)
- Workflow steps — stable, in skill files

## Fallback

If the fetch script fails, use the Shengwang doc site URL directly:
```
https://doc.shengwang.cn/doc/{product}/{platform}/{path}
```
