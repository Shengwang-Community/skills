# AGENTS.md

This repository contains AI agent skills for Shengwang platform integration,
following the [Agent Skills](https://agentskills.io) open standard.

## How to Use

> **CRITICAL FIRST STEP:** Start from `skills/voice-ai-integration/SKILL.md` and let it route to the correct product module. The bundled doc indexes under `references/doc-index/` (`docs.index.md`, `docs.index.json`, shards) are fallback lookup aids — use them only when the product module still needs external documentation.
> `fetch-docs.sh` and `scripts/build-doc-index.py` are maintainer tools for refreshing those committed artifacts, not runtime steps for every session.

1. Start from [skills/voice-ai-integration/SKILL.md](skills/voice-ai-integration/SKILL.md) — it is the root entry point
2. Let SKILL.md route the request to the correct product module (or ask one clarifying question if the product is unclear)
3. Follow that product module's workflow first
4. Only use `docs.index.md`, product shards, or `doc-fetching.md` when the product module still needs external documentation lookup

## Repository Structure

```
scripts/
├── fetch-docs.sh              # Download sitemap + rebuild doc-index
├── build-doc-index.py         # Generate all index files from sitemap
├── ab-test-doc-index.py       # Token-based benchmark (56 cases)
├── llm-eval-doc-index.py      # LLM end-to-end eval (72 cases, needs OPENAI_API_KEY)
└── validate-skills.sh         # Link and frontmatter validation
skills/
└── voice-ai-integration/         # The skill (start here)
    ├── SKILL.md                   # Entry point and router (only SKILL.md in the repo)
    └── references/                # All product modules and shared knowledge
        ├── doc-fetching.md            # Doc fetching guide
        ├── doc-index/                 # Prebuilt doc indexes (generated by build-doc-index.py)
        │   ├── docs.index.md          # Agent-readable product catalog with URIs
        │   ├── docs.index.json        # Full machine-readable index
        │   ├── api-reference.json     # SDK class/interface docs (separate)
        │   └── shards/               # Per-product and per-platform shards
        ├── general/               # Credentials, REST auth (shared)
        ├── conversational-ai/     # ConvoAI
        ├── rtc/                   # RTC SDK
        ├── rtm/                   # RTM
        ├── cloud-recording/       # Cloud Recording
        └── token-server/          # Token generation
```

## Documentation Access

Skills provide behavioral guidance and workflow. Up-to-date API documentation is fetched
directly via HTTP from `doc-mcp.shengwang.cn` — no MCP server setup required.
See [references/doc-fetching.md](skills/voice-ai-integration/references/doc-fetching.md) for doc fetching patterns.

## Rules

- **FIRST ACTION on every session:** start from `skills/voice-ai-integration/SKILL.md` and the routed product module
- Always start from `skills/voice-ai-integration/SKILL.md`
- Never hardcode credentials — use environment variables
- Use local module docs first; only use the bundled indexes and remote fetch when the module still needs external documentation lookup
- When fetch fails, use local references + fallback URLs, and tell the user to verify
